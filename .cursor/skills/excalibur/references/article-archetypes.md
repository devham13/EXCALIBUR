# Excalibur — archetypes статей

Маппинг intent из `11-blog-topics.md` → структура (режим A/B из контракта).

## A — Новость / кейс / разбор

**⚠️ Utility-only блог: режим A не публикуем.** Оставлено для справки.

**Когда:** news_case, informational с событием, trend.

**Скелет HTML:**

1. Hook + direct answer
2. Что произошло / контекст (факты + даты)
3. Что это значит читателю (аудитория из audience-map)
4. Риски / ограничения
5. Что попробовать (практика)
6. FAQ
7. Что дальше + CTA

## B — Инструкция / гайд

**Когда:** how_to, parent_guide, checklist.

**Скелет:**

1. Hook + результат после прохождения
2. Кому подходит / prerequisites
3. Шаги (`<ol>`) — 5–9 нумерованных шагов. **Обязательно** добавить схему-цепочку процесса (`<blockquote>` со стрелками `→`) перед началом шагов и минимум 1 изображение-иллюстрацию (или ТЗ-плейсхолдер `<img src="..." alt="..." />`) в середине гайда с подписью `<i>`.
4. Типичные ошибки
5. Инструменты / alternatives (без VPN/обхода)
6. FAQ
7. Что дальше + CTA

## Comparison / Обзоры

**Когда:** comparison в archetype темы.

- Обязательно внедрить **сравнительную таблицу (`<table>`)** во 2-й или 3-й секции с 4-5 параметрами сравнения.
- Split-логика или side-by-side сравнение в тексте.
- Критерии сравнения из research.
- **Экспертный вердикт** в конце сравнения через оформленный блок `<blockquote>` с жирными акцентами, дающий объективный выбор для разных сегментов ЦА.
- Verdict без «лучший в мире».

## Объём по секциям (8500–9500 знаков)

| Блок | ~% |
|------|-----|
| Hook + context | 15% |
| Main H2 sections (5–8) | 70% |
| FAQ | 10% |
| Что дальше | 5% |

## CTA placement

- Мягкий CTA внутри одной H2-секции (блок 4 контракта)
- Основной CTA в «Что дальше»
- Max 3 упоминания офера на статью
