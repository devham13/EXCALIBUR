ИИ в Cursor умеет писать тексты, но черновик в WordPress вы всё равно копируете руками - REST API пугает, а "AI-плагины" обещают магию без контроля. Официальный WordPress MCP Adapter переводит возможности сайта в tools для агента: discover abilities, чтение, черновик с вашей проверкой. За 30-60 минут на self-hosted WordPress 6.9+ вы установите adapter, создадите отдельного пользователя с Application Password и подключите Cursor - без публикации в один клик.

TL;DR / Быстрый инсайт: WordPress MCP - мост между Abilities API и ИИ-клиентом. Путь: MCP Adapter → Application Password → .cursor/mcp.json → discover → read-only → draft → проверка человеком. Не давайте агенту admin; публикация - после GEO-чеклиста.

WordPress MCP - управление сайтом через AI по стандарту MCP (Model Context Protocol, открытая спецификация Anthropic с ноября 2024): агент видит discoverable tools, а не сырой REST. Abilities API с WP 6.9 описывает действия сайта в REST namespace wp-abilities/v1; MCP Adapter (v0.5.0, релиз апрель 2026) регистрирует server mcp-adapter-default-server на /wp-json/mcp/mcp-adapter-default-server. Пакет входит в инициативу AI Building Blocks for WordPress. MCP в Cursor ещё не настроен - начните с подключения MCP в Cursor; эта статья - следующий шаг именно для WordPress. Ниже - self-hosted; на WordPress.com MCP работает через OAuth в настройках аккаунта (планы Personal и выше).

1. Разберите, зачем бизнесу WordPress MCP и Abilities API

Abilities API - каталог разрешённых действий сайта. MCP Adapter - переводчик для AI: клиент вызывает tools, adapter проверяет права. В ядре 6.9 три read-abilities: site-info, user-info, environment-info. Write появляется, если плагин пометил ability как meta.mcp.public => true.

Сценарий Запрос агенту Human-in-the-loop
Черновик Draft поста по брифу Вы правите и проходите GEO перед publish
Аудит 10 старых постов в рубрике Вы решаете, что обновлять
Мета Alt-тексты для вложений Медиа публикуете сами на старте

Делайте: read-only и audit первыми. Не делайте: auto-publish - менее 2,5% сложных задач ИИ закрывает без человека; draft + одобрение - норма.

2. Проверьте требования: WordPress 6.9+, HTTPS, отдельный пользователь

Abilities API - с WordPress 6.9 "Gene" (2 декабря 2025). MCP Adapter v0.5.0 требует PHP 7.4+ и WP 6.8+, но abilities in core - только 6.9+. На продакшене PHP 8.0+, HTTPS обязателен: Application Password и remote bridge не работают по HTTP. На машине с Cursor нужен Node.js для команды npx @automattic/mcp-wordpress-remote - это прокси между редактором и вашим сайтом.

Application Password - ключ API, который отзывается без смены пароля wp-admin. Создайте mcp-editor с ролью Editor или Author, не Administrator.

  1. Шаг 1: Проверьте WP 6.9+ (Обновления или /wp-json/).
  2. Шаг 2: HTTPS валиден, REST /wp-json/ не отдаёт 403.
  3. Шаг 3: Новый пользователь Editor/Author для MCP.
  4. Шаг 4: Application Password "Cursor MCP" - скопируйте с пробелами, показывается один раз.
  5. Шаг 5: Сохраните пароль offline, не в чат Cursor.

Делайте: least privilege. Не делайте: admin для MCP.

3. Установите MCP Adapter и проверьте endpoint

Скачайте v0.5.0 с GitHub WordPress/mcp-adapter или composer require wordpress/mcp-adapter. Активируйте в wp-content/plugins/mcp-adapter.

Схема: adapter → Application Password → mcp.json → перезапуск Cursor → discover → read-only → draft → GEO → publish
Транспорт Когда Как
HTTP remote Продакшен + Cursor на ПК @automattic/mcp-wordpress-remote + WP_API_URL
STDIO Локальная dev WP-CLI: wp mcp-adapter serve --server=mcp-adapter-default-server

Проверка: откройте https://ваш-сайт.ru/wp-json/mcp/mcp-adapter-default-server в браузере или curl. Ожидайте auth challenge или JSON-ответ, не 404. При 404 - плагин не активен или сбросились постоянные ссылки (Настройки → Постоянные ссылки → Сохранить). При необходимости в папке плагина выполните composer install, если ставили через Composer.

Делайте: official adapter на новых инсталляциях. Не делайте: устаревший automattic/wordpress-mcp вместо него.

4. Настройте Cursor: mcp.json и remote bridge

Tools & MCP → Add Custom MCP → .cursor/mcp.json (как в базовом гайде MCP):

{
  "mcpServers": {
    "wordpress-mcp-server": {
      "command": "npx",
      "args": ["-y", "@automattic/mcp-wordpress-remote@latest"],
      "env": {
        "WP_API_URL": "https://your-site.com/wp-json/mcp/mcp-adapter-default-server",
        "WP_API_USERNAME": "mcp-editor",
        "WP_API_PASSWORD": "xxxx xxxx xxxx xxxx xxxx xxxx"
      }
    }
  }
}

Замените your-site.com, mcp-editor и пароль на свои значения. Пароль вставляйте с пробелами, как показал WordPress. После правки - полный перезапуск Cursor (не только reload окна). В Tools & MCP статус зелёный; появятся meta-abilities: discover-abilities, get-ability-info, execute-ability. Если MCP не стартует - проверьте Node, путь npx и логи в панели Cursor; детали базовой диагностики - в гайде по MCP.

Делайте: mcp.json в .gitignore. Не делайте: коммит пароля.

5. Выполните первые команды агенту: discover, read, draft

  1. Discover: "Список WordPress abilities через MCP".
  2. Read-only: core/get-site-info - ответ с вашего домена.
  3. Аудит: старые посты рубрики - только чтение.
  4. Draft: "Черновик по тезисам, без publish" - проверка в wp-admin.
  5. GEO: GEO-оптимизация перед публикацией.

Cursor запросит подтверждение перед вызовом tool - это ваш human-in-the-loop на уровне клиента. Пустой список write-tools часто значит: abilities сторонних плагинов не помечены meta.mcp.public => true в коде разработчика. Realistic MVP на старте - core read abilities плюс черновик там, где abilities provider уже exposed write. После правки черновика прогоните материал по GEO-чеклисту: структура H2, факты, блоки для нейропоиска.

Делайте: подтверждайте write в Cursor. Не делайте: "опубликуй на главную".

6. Настройте безопасность и устраните 401, 403, пустые tools

MCP-клиент авторизуется как залогиненный WP-пользователь; каждый ability проверяет permission_callback. Wordfence, iThemes и аналоги часто режут /wp-json/ - добавьте whitelist для /wp-json/mcp/. Ошибка 401: неверный Application Password (проверьте пробелы), истёкший ключ или HTTP вместо HTTPS. 403: недостаточная роль (Author не правит чужие посты) или REST заблокирован. Пустые tools после discover: abilities не public для MCP.

  • Read-only первая неделя.
  • Отзыв пароля: Application Passwords → удалить.
  • Логи security plugin для mcp-editor.

Делайте: регламент draft → человек → GEO → publish. Не делайте: write от admin.

7. Пройдите чеклист перед продакшеном

  • WP 6.9+, PHP 8.0+, HTTPS.
  • REST и MCP endpoint доступны.
  • Adapter v0.5.0 активен.
  • Отдельный user, Application Password offline.
  • mcp.json корректен, Cursor перезапущен.
  • Discover и read-only OK.
  • Черновик проверен в wp-admin.
  • GEO перед publish.
  • План отзыва пароля.
  • Нет admin в MCP-конфиге.

Частые вопросы

Что такое WordPress MCP Adapter?

Официальный пакет инициативы AI Building Blocks: мост между Abilities API и MCP. После активации регистрирует default-server на /wp-json/mcp/mcp-adapter-default-server и три meta-abilities - discover-abilities, get-ability-info, execute-ability. AI-клиент видит tools, adapter выполняет действия с проверкой прав WordPress.

Какие версии WordPress нужны?

Минимум 6.9 для Abilities в ядре. Adapter формально на 6.8+ с отдельным plugin, но для бизнеса ставьте 6.9+.

Как подключить WordPress MCP к Cursor?

Adapter на сайте → Application Password → блок в .cursor/mcp.json с @automattic/mcp-wordpress-remote → перезапуск Cursor. База MCP - отдельный гайд.

Чем MCP отличается от REST API?

REST - ручные HTTP из кода. MCP - discover tools, затем execute: агент не пишет curl на каждый endpoint.

Безопасно ли давать AI write-доступ?

Только с dedicated user, least privilege и draft без auto-publish. На WordPress.com write-операции требуют подтверждения в UI. Self-hosted: read-only первую неделю, write только в черновики, финальная публикация - вручную в wp-admin после GEO-проверки.

Почему tools пустые или 401?

Неверный пароль (нужны пробелы), HTTP вместо HTTPS, security блокирует /wp-json/, abilities без meta.mcp.public.

WordPress.com vs self-hosted MCP?

.com: OAuth, настройки в аккаунте. Self-hosted: mcp-adapter + Application Password + npx bridge. Auth не смешивайте.
